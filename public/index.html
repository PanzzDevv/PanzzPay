Sistem Gateway Pembayaran QRIS Otomatis PanzzPay

PanzzPay berfungsi mengecek dan mencatat pembayaran QRIS secara otomatis hanya dengan membaca notifikasi handphone Anda (via aplikasi Android Forwarder). Nikmati integrasi REST API tanpa biaya admin per transaksi!

// ALUR KERJA SISTEM OTOMATIS:
1. QRIS Dinamis Sistem membuat QRIS dengan nominal unik (Contoh: Rp 10.338).
2. Pembeli Transfer Pembeli men-scan QRIS via DANA, ShopeePay, OVO, atau m-Banking.
3. Notifikasi HP HP Anda menerima SMS / Pop-up notifikasi dana masuk.
4. Webhook Forwarder App Forwarder di HP meneruskan teks notifikasi ke Server PanzzPay.
5. Status LUNAS Server me-validate nominal & otomatis mengubah invoice ke PAID!
SERVER BASE URL: http://localhost:3000

Pilih aplikasi bank atau e-wallet tempat Anda menerima pembayaran (Contoh: DANA, OVO, ShopeePay, GoBiz, m-BCA, BRImo, Livin).

            API DOCUMENTATION FOR DEVELOPERS
            

              
              Dokumentasi Integrasi PanzzPay REST API
            

            
Gunakan endpoint di bawah ini untuk menghubungkan PanzzPay langsung ke backend website/aplikasi Anda sendiri.

          


          

            
            
            

              

                POST
                /api/qris/generate
              

              
Membuat QRIS Dinamis lengkap dengan total nominal tagihan dan kode unik.


              
PARAMETER (JSON Body):

              
    
                
  • base_amount (number, wajib): Harga dasar barang.
    • 
                
  • api_key (string, opsional): API Key unik merchant Anda dari Portal PanzzPay.
    • 
              


              
CONTOH cURL:

              
curl -X POST http://localhost:3000/api/qris/generate \
  -H 'Content-Type: application/json' \
  -H 'X-API-KEY: pz_live_your_merchant_key' \
  -d '{"base_amount": 20000, "auto_unique": true}'

            


            
            

              

                POST
                /api/webhook/callback?token=pz_wh_xxx
              

              
Menerima lemparan notifikasi HP dan mencocokkan status invoice ke PAID.


              
FORMAT WEBHOOK PAYLOAD:

              
{
  "message": "Pembayaran masuk Rp 10.338 dari ShopeePay"
}


              
CONTOH cURL:

              
curl -X POST http://localhost:3000/api/webhook/callback?token=pz_wh_your_token \
  -H 'Content-Type: application/json' \
  -d '{"message": "Pembayaran Rp 10.338 diterima"}'

            


            
            

              

                GET
                /api/invoices/:id
              

              
Mengecek status terkini suatu invoice (PENDING, PAID, atau EXPIRED).


              
RESPON JSON SUKSES:

              
{
  "ok": true,
  "invoice": {
    "id": "INV-01ABC",
    "total_amount": 10338,
    "status": "PAID"
  }
}
